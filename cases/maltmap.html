몰트맵 - 상품 검색·가격 비교 서비스

탐색형 서비스는 데이터가 많을수록 좋아 보이지만, 사용자가 원하는 상품을 빠르게 좁히지 못하면 이탈합니다. 몰트맵 사례는 상품 정보, 가격대, 관심 상품을 하나의 탐색 경험으로 묶는 방식에 집중했습니다.

MM
유형
웹 서비스 / 상품 탐색
범위
검색, 필터, 관심 상품, 가격 비교
기간 예시
4~6주 MVP
핵심 가치
탐색 피로를 줄이는 정보 구조

문제 정의

가격 비교형 서비스에서 사용자는 “가장 싼 상품”만 찾는 것이 아니라, 조건에 맞는 상품을 빠르게 좁히고 나중에 다시 비교할 수 있어야 합니다. 데이터가 단순 리스트로만 제공되면 사용자는 여러 페이지를 오가며 같은 비교를 반복하게 됩니다.

접근 방식

01검색 의도 분리

상품명 검색, 카테고리 탐색, 가격대 비교를 서로 다른 진입 흐름으로 정리했습니다.

02비교 기준 고정

가격, 판매처, 용량, 평점처럼 반복 확인하는 정보를 카드 안에 일관되게 배치했습니다.

03관심 상품 저장

당장 구매하지 않는 사용자가 다시 돌아와 비교할 수 있도록 관심 목록을 설계했습니다.

구현 범위

  • 상품 데이터 목록과 상세 화면 구성
  • 카테고리, 가격대, 키워드 기반 필터
  • 관심 상품 저장 및 비교 후보 관리
  • 판매처별 가격 후보 노출 UI
  • 관리자가 상품 정보를 등록, 수정, 비활성화할 수 있는 기본 구조

산출물

탐색 화면검색, 필터, 정렬, 결과 카드
비교 구조가격 후보, 상품 속성, 관심 목록
관리자상품 등록, 데이터 보정, 노출 상태 관리

핵심 포인트

  • 검색 서비스는 데이터보다 필터 기준이 먼저 정리되어야 합니다.
  • 관심 상품은 재방문과 비교 행동을 만드는 중요한 장치입니다.
  • 초기에는 자동 수집보다 관리 가능한 데이터 구조가 더 중요할 수 있습니다.
