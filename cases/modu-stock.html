모두의 주식 - 관심 종목 기반 콘텐츠 요약

금융 콘텐츠는 양이 많고 갱신 속도가 빠릅니다. 사용자는 모든 글을 읽기보다 내 관심 종목과 관련된 핵심만 먼저 보고 싶어합니다. 이 사례는 콘텐츠를 단순 축적이 아니라 판단을 돕는 화면 흐름으로 바꾸는 방식에 집중했습니다.

유형
콘텐츠 앱 / 요약 서비스
범위
요약 카드, 관심 종목, 콘텐츠 추천
기간 예시
4~7주 MVP
핵심 가치
긴 콘텐츠를 읽기 전 판단 정보 제공

문제 정의

투자 관련 콘텐츠는 제목만으로는 중요한 정보를 판단하기 어렵고, 본문은 길어 모바일에서 읽기 부담스럽습니다. 특히 관심 종목이 정해져 있는 사용자는 전체 콘텐츠 피드보다 “내 종목과 관련된 변화”를 먼저 보고 싶어합니다.

그래서 핵심 목표를 콘텐츠 생산량 확대가 아니라, 콘텐츠를 요약하고 사용자 관심사와 연결해 소비 시간을 줄이는 것으로 정의했습니다.

접근 방식

01요약 카드 설계

본문 전체를 보기 전에 핵심 이슈, 관련 종목, 긍정/부정 포인트를 카드로 확인하게 했습니다.

02관심 종목 연결

사용자가 저장한 종목과 관련된 콘텐츠를 우선 노출하는 구조로 피드 우선순위를 잡았습니다.

03관리 가능한 콘텐츠 구조

관리자가 콘텐츠를 등록하고 요약 결과를 검수할 수 있는 운영 흐름을 고려했습니다.

구현 범위

  • 콘텐츠 목록, 상세, 요약 카드 UI
  • 관심 종목 등록과 종목별 콘텐츠 필터
  • 관련 키워드, 종목 태그, 콘텐츠 추천 흐름
  • AI 요약 결과를 검토하고 수정할 수 있는 관리자 구조
  • 사용자가 저장한 콘텐츠를 다시 확인할 수 있는 북마크 흐름

산출물

사용자 화면요약 피드, 상세, 관심 종목, 저장 목록
AI 연동요약 프롬프트, 결과 검수, 키워드 추출
관리자콘텐츠 등록, 요약 수정, 노출 관리

핵심 포인트

  • AI 요약은 결과를 그대로 노출하기보다 검수 흐름이 필요합니다.
  • 관심 종목 같은 개인화 기준이 있어야 콘텐츠 피드가 유용해집니다.
  • 콘텐츠 서비스는 사용자 화면과 관리자 화면을 함께 설계해야 운영이 가능합니다.
