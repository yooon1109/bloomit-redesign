AI/OCR feature

얼마지? - 상품 택 OCR 가격 탐색

사용자가 상품명을 직접 입력하지 않아도 가격을 찾아볼 수 있게 하려면 촬영, 인식, 보정, 검색어 생성, 결과 노출까지 이어지는 기능 흐름이 필요합니다. 얼마지 사례는 AI/OCR 기능을 실제 사용자 행동 안에 자연스럽게 넣는 방식에 집중했습니다.

유형
AI/OCR 기능 / 가격 탐색
범위
이미지 업로드, OCR, 검색어 생성, 결과 후보
기간 예시
3~6주 PoC/MVP
핵심 가치
입력 부담을 줄이는 AI 기능 설계

문제 정의

가격 탐색 서비스에서 가장 큰 진입 장벽은 정확한 상품명과 모델명을 사용자가 직접 입력해야 한다는 점입니다. 특히 의류, 잡화, 중고 상품처럼 택이나 라벨에 정보가 있는 경우 사용자는 사진은 쉽게 찍을 수 있지만, 인식 결과가 틀리면 검색 품질이 바로 떨어집니다.

따라서 핵심은 OCR 자체가 아니라, 인식 결과를 사용자가 수정할 수 있고 가격 검색에 적합한 키워드로 바꾸는 흐름을 만드는 것입니다.

접근 방식

01촬영 가이드

택이 잘 보이도록 촬영 영역과 안내 문구를 배치해 인식 실패 가능성을 줄였습니다.

02OCR 결과 보정

브랜드, 상품명, 모델명, 사이즈 후보를 나누고 사용자가 확인/수정할 수 있게 했습니다.

03가격 후보 노출

생성된 검색어로 후보 가격을 보여주고, 사용자가 적합한 결과를 선택할 수 있게 했습니다.

구현 범위

  • 이미지 촬영 또는 업로드 화면
  • OCR API 연동과 결과 파싱
  • 브랜드, 상품명, 모델명 후보 분리
  • 검색 키워드 자동 생성 및 사용자의 직접 수정 흐름
  • 온라인 가격 후보 목록과 상세 이동
  • 인식 실패, 결과 없음, 네트워크 오류에 대한 예외 화면

산출물

사용자 플로우촬영, 인식, 보정, 가격 후보 확인
AI/OCR 연동이미지 전송, 결과 파싱, 키워드 생성
검증 기준인식 성공률, 수정 UX, 예외 케이스 체크

핵심 포인트

  • AI 기능은 정확도보다 사용자 수정 흐름까지 포함해야 제품이 됩니다.
  • PoC 단계에서는 성공 케이스와 실패 케이스를 함께 검증해야 합니다.
  • OCR 결과를 그대로 쓰지 말고 검색 목적에 맞게 구조화해야 합니다.
