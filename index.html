Process

각 단계마다 확인 가능한 산출물을 남깁니다.

외주 프로젝트는 중간 확인이 없을수록 위험합니다. Bloomit은 말로만 합의하지 않고 기능표, 화면 흐름, 테스트 링크로 진행 상황을 공유합니다.

  1. 01

    상담

    목표, 사용자, 예산, 일정, 참고 서비스를 정리합니다.

    요구사항 메모
  2. 02

    범위 확정

    필수 기능과 후순위 기능을 나눠 견적 기준을 만듭니다.

    기능 범위표
  3. 03

    화면 설계

    사용자, 관리자, 데이터 흐름을 화면 단위로 정리합니다.

    화면 흐름도
  4. 04

    구현

    확인 가능한 단위로 개발하고 테스트 링크를 공유합니다.

    테스트 빌드
  5. 05

    검수·출시

    실제 사용 흐름으로 오류를 확인하고 배포를 준비합니다.

    체크리스트

Best fit

이런 프로젝트와 잘 맞습니다.

모든 외주를 무조건 받기보다, Bloomit이 강점을 낼 수 있는 상황을 분명히 제시합니다. 의뢰자도 상담 전에 맞는 파트너인지 빠르게 판단할 수 있습니다.

잘 맞는 의뢰

  • 아이디어는 있지만 기능 범위가 아직 정리되지 않은 경우
  • MVP를 먼저 만들고 시장 반응을 확인하고 싶은 경우
  • 앱, 웹, 서버, 관리자까지 한 흐름으로 맡기고 싶은 경우
  • AI/OCR 기능을 서비스 안에 실험적으로 붙이고 싶은 경우

먼저 조율이 필요한 의뢰

  • 요구사항 없이 바로 개발 착수를 원하는 경우
  • 기획 변경 가능성이 크지만 일정과 예산은 고정인 경우
  • 출시 이후 운영자 화면이나 유지보수를 고려하지 않는 경우
  • 법적 검토가 필요한 회색 영역 서비스인 경우

Project guide

견적은 기능 수보다 불확실성에서 커집니다.

그래서 초기 상담에서는 “얼마예요?”보다 “무엇을 먼저 만들면 되는지”를 함께 정리합니다. 아래 기준은 상담 전 방향을 잡기 위한 가이드입니다.

랜딩·회사 홈페이지 2~4주

서비스 소개, 문의 접수, 반응형 화면, SEO 기본

MVP 웹·앱 4~8주

핵심 사용자 흐름, 로그인, 데이터 저장, 관리자 기본

플랫폼·운영 시스템 8주 이상

권한, 결제/예약/정산, 검색, 운영 대시보드

FAQ

문의 전에 자주 확인하는 것들

기획안이 없어도 문의할 수 있나요?

가능합니다. 만들고 싶은 서비스, 참고 사이트, 꼭 필요한 기능만 있어도 범위를 함께 정리할 수 있습니다.

디자인만 또는 개발만 맡길 수 있나요?

가능하지만, 프로젝트 목적에 따라 기획-디자인-개발을 함께 보는 편이 결과물의 완성도가 높습니다.

출시 후 유지보수도 가능한가요?

수정 범위와 운영 방식에 따라 월 단위 또는 건별 유지보수를 협의할 수 있습니다.

AI 기능은 어느 단계에서 검토하나요?

초기 상담에서 기술 가능성, 비용, 정확도 리스크를 먼저 확인하고 PoC 또는 MVP 범위로 나눠 진행합니다.
